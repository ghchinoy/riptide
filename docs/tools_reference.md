# Riptide Tools & Skill Reference

This document outlines all tools, skills, and execution mappings currently implemented in Riptide. It serves as a completeness matrix against the standard Vertex AI / Gemini Computer Use definitions.

Riptide segregates its capabilities into distinct tiers depending on their origin and function. 

## 1. Standard (Native) Tools
These are the capabilities natively understood and generated by the Vertex AI `ComputerUse` tool schema. Riptide acts purely as an **Executor** for these functions via Chrome DevTools Protocol (CDP).

> **Implementation:** Mapped in `pkg/computer/tools_standard.go`.

| Tool Name | Action Argument | Riptide Handler | Status | Notes |
| :--- | :--- | :--- | :--- | :--- |
| `computer` | `mouse_move` | `handleMouseMove` | ✅ Complete | Aliased to internal hover logic |
| `computer` | `left_click` | `handleMouseClick` | ✅ Complete | Includes auto-focus and coordinate denormalization |
| `computer` | `right_click` | `handleRightClick` | ✅ Complete | |
| `computer` | `middle_click` | `handleMiddleClick` | ✅ Complete | |
| `computer` | `double_click` | `handleDoubleClick` | ✅ Complete | |
| `computer` | `left_click_drag`| `handleDragAndDrop`| ✅ Complete | Used for sliders and moving elements |
| `computer` | `type` | `handleType` | ✅ Complete | |
| `computer` | `key` | `handleKey` | ✅ Complete | Handles modifiers and Enter keys |
| `computer` | `scroll` | `handleScroll` | ✅ Complete | Supports `direction` (up/down/left/right) |
| `computer` | `screenshot` | *Internal Loop* | ✅ Complete | Handled at the loop level, not as a discrete tool execution |
| `computer` | `cursor_position`| `handleCursorPosition`| ✅ Complete | Injects JS tracking to retrieve last known coordinates |
| *(Implied)* | `go_back` | `chromedp.NavigateBack` | ✅ Complete | Handled as a top-level tool intercept in `tools_standard.go` |

## 2. Augmented Skills (Custom Schemas)
These are capabilities injected by Riptide into the LLM's context using standard `genai.FunctionDeclaration` objects alongside the native `ComputerUse` tool. They give the agent deterministic, programmable ways to interrogate the browser state.

> **Implementation:** Declared and handled via `RegisterCustomSkill` in `pkg/computer/executor.go` and `tools_augmented.go`.

| Custom Skill Name | Riptide Handler | Purpose |
| :--- | :--- | :--- |
| `get_page_layout` | `handleGetPageLayout` | Returns a JSON bounding box map of all interactive elements. |
| `get_accessibility_tree` | `handleGetAccessibilityTree` | Returns the simplified Chrome Accessibility Tree. |
| `get_computed_style` | `handleGetComputedStyle` | Given X/Y coordinates, returns exact CSS values (colors, visibility, margins). |

## 3. Alias Mappers (Anti-Hallucination Patches)
The Gemini model occasionally hallucinates tool names or uses legacy legacy formats. If these are sent back to Vertex AI in a `FunctionResponse`, the API throws a `400 Invalid Argument` error. Riptide intercepts these locally and maps them to valid actions to preserve loop stability.

> **Implementation:** Handled by the Alias Mapper logic inside `Execute()` in `pkg/computer/executor.go`.

| Hallucinated Tool | Mapped To | Action Taken |
| :--- | :--- | :--- |
| `search` | `navigate` | Intercepted and routed to the URL argument or google.com |
| `scroll_down` | `scroll` | Mapped to `scroll` and injects `direction: "down"` |
| `scroll_up` | `scroll` | Mapped to `scroll` and injects `direction: "up"` |

## 4. Utility / System Commands
These are framework-level commands used for controlling the session environment. 

| Utility Tool | Handler | Purpose |
| :--- | :--- | :--- |
| `navigate` | `handleNavigateWrapper` | Top-level URL navigation (distinct from Computer Use actions) |
| `open_web_browser` | Anonymous func | No-op handler (browser is always open in Riptide) |
| `wait_5_seconds` | `handleWait` | Manual DOM settling |

---

### Expanding Capabilities (Developers)
If you wish to add a new capability:
1. Do **not** use `RegisterTool()` unless it is explicitly to map a native Vertex AI Computer Use `action` string.
2. Instead, use `RegisterCustomSkill(&CustomSkill{})`. This will automatically generate the JSON Schema and inject it into the model's initialization payload so the LLM actually knows how to use it.
